# Contribution au site OPEO

Merci de votre intérêt pour contribuer au site officiel d'OPEO Consulting!

## À propos

Ce site est un projet officiel d'OPEO Consulting. Les contributions sont gérées par l'équipe interne.

## Comment contribuer

### Signaler un bug
1. Vérifiez que le bug n'a pas déjà été rapporté
2. Décrivez le comportement attendu et le comportement observé
3. Fournissez des détails sur votre environnement (navigateur, OS, etc.)

### Proposer une amélioration
1. Décrivez l'amélioration proposée
2. Expliquez pourquoi cette amélioration serait utile
3. Listez les exemples d'implémentation si possible

### Soumettre des modifications

Pour les employés d'OPEO:
1. Créer une branche depuis `main`
2. Commiter vos modifications avec des messages clairs
3. Soumettre une pull request
4. Attendre la review et les approbations

Pour les contributeurs externes:
- Les contributions externes doivent être approuvées par l'équipe OPEO

## Standards de code

### HTML
- Utiliser une indentation cohérente (2 espaces)
- Respecter la structure sémantique HTML5
- Ajouter des commentaires pour les sections complexes

### CSS
- Utiliser des classes pour le styling (pas de styles inline)
- Maintenir la cohérence avec les styles existants
- Optimiser les performances CSS

### JavaScript
- Écrire du code lisible et maintenable
- Ajouter des commentaires pour la logique complexe
- Tester sur les navigateurs principaux

## Processus de review

1. Au moins une review est requise
2. Les changements doivent passer les tests (s'il y en a)
3. Les merge conflicts doivent être résolus
4. Merge via la branche `main`
